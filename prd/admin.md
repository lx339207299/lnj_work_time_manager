# 后台管理系统 (Admin) 需求文档 (PRD)

> **版本**: V1.0  
> **面向对象**: 超级管理员 / 运维人员 / 财务  
> **定位**: 全局管理、数据维护、系统配置

---

## 1. 核心目标

建立一个独立于小程序业务之外的**全局管控平台**，赋予管理员“上帝视角”，用于：
1.  **管理用户与组织**：解决账号问题、组织归属问题。
2.  **数据监管**：查看全局业务数据，处理异常情况。
3.  **安全审计**：记录关键操作，确保系统安全。

---

## 2. 功能清单 (Feature List)

### Phase 1: 核心基础 (已完成骨架)

| 优先级 | 功能模块 | 功能点 | 描述 |
| :--- | :--- | :--- | :--- |
| **P0** | **鉴权** | 登录 | 支持手机号/密码登录；Token 自动续期。 |
| **P0** | **鉴权** | 退出登录 | 清除本地状态，重定向至登录页。 |
| **P0** | **概览** | 仪表盘 (Dashboard) | 展示系统核心指标：注册用户数、活跃组织数、今日新增工时、今日活跃项目数。 |

### Phase 2: 用户与组织管理 (High Priority)

| 优先级 | 功能模块 | 功能点 | 描述 |
| :--- | :--- | :--- | :--- |
| **P1** | **用户管理** | 用户列表 | 分页展示所有注册用户。支持按手机号、昵称、注册时间搜索。 |
| **P1** | **用户管理** | 用户详情 | 查看用户基本信息、所属组织、参与项目列表。 |
| **P1** | **用户管理** | 账号封禁/解封 | 针对违规用户进行封禁 (`isLocked=true`)，封禁后用户无法登录小程序。 |
| **P1** | **用户管理** | 密码重置 | 强制重置用户密码（用于用户忘记密码且无法自行找回的紧急情况）。 |
| **P1** | **组织管理** | 组织列表 | 分页展示所有组织。支持按名称、创建人搜索。 |
| **P1** | **组织管理** | 组织详情 | 查看组织下的成员列表、项目列表、工时概览。 |
| **P2** | **组织管理** | 组织下架 | 逻辑删除违规组织（保留数据但不可访问）。 |

### Phase 3: 业务数据监管 (Medium Priority)

| 优先级 | 功能模块 | 功能点 | 描述 |
| :--- | :--- | :--- | :--- |
| **P2** | **项目监管** | 全局项目列表 | 跨组织查看所有项目。支持按状态（进行中/已结束）筛选。 |
| **P2** | **项目监管** | 强制归档 | 对长期无活动或违规项目进行强制归档/关闭。 |
| **P2** | **工时监管** | 全局工时查询 | 按日期范围、用户、组织筛选工时记录。 |
| **P3** | **工时监管** | 数据修正 | **(慎用)** 允许管理员修正错误的工时数据（需记录严格的审计日志）。 |

### Phase 4: 系统运维 (System Ops)

| 优先级 | 功能模块 | 功能点 | 描述 |
| :--- | :--- | :--- | :--- |
| **P2** | **审计** | 操作日志 | 查看管理员的操作记录（谁、在什么时间、对哪个对象、做了什么操作）。 |
| **P3** | **配置** | 系统参数 | 配置全局参数（如：默认工时类型、通知开关等，视后续需求而定）。 |

---

## 3. 权限与角色

*   **超级管理员 (Super Admin)**: 拥有所有权限，可管理管理员账号。
*   **普通管理员 (Admin)**: 拥有除“创建管理员”外的所有业务管理权限。
*   *(V1 阶段仅实现单一的 Admin 角色)*

---

## 4. 非功能需求

1.  **响应速度**: 列表加载时间 < 1s。
2.  **安全性**: 所有管理接口必须校验 `systemRole`；敏感操作（如重置密码）需二次确认。
3.  **审计**: 关键数据的增删改查必须记录 `SystemLog`。
